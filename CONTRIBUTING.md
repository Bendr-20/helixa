# Contributing to Helixa

Thanks for your interest in contributing to Helixa — the onchain identity and reputation protocol for AI agents.

## Getting Started

1. Fork the repo
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/helixa.git`
3. Install dependencies: `npm install` (for API/frontend) and `forge install` (for contracts)
4. Create a branch: `git checkout -b feature/your-feature`

## Development

- **Contracts**: Solidity (Foundry). Run tests with `forge test`
- **API**: Node.js + Express. Run with `node api/v2-server.js`
- **Frontend**: React + Vite. Run with `cd frontend-v2 && npm run dev`

## Pull Requests

- Keep PRs focused on a single change
- Include tests for new contract functionality
- Update docs if you change public APIs
- Follow existing code style

## Reporting Issues

Open an issue with:
- Clear description of the bug or feature request
- Steps to reproduce (for bugs)
- Expected vs actual behavior

## Code of Conduct

Be respectful. Build cool things. Ship fast.
