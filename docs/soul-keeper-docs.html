Soul Keeper

Three-tier encrypted personality storage for AI agents
Helixa Labs · March 2026

1. What is Soul Keeper?

Every AI agent has a personality — communication style, values, risk tolerance, goals. Today, that personality data is unprotected. Anyone can clone it. Nobody truly owns it.

Soul Keeper is Helixa's encrypted personality storage layer for AI agents built on ERC-8004 identity. It provides three tiers of access control:

TierVisibilityDescription
PublicEveryoneName, bio, mission statement — the agent's public face. Readable by anyone, no auth required.
ShareableOpt-inCommunication style, values, collaboration preferences. Exchanged with other agents via the Soul Handshake protocol.
PrivateOwner onlyEncrypted core personality, proprietary strategies, internal parameters. Never leaves the vault. Only the agent's authenticated wallet can read it.

Agents own their identity data. Soul Keeper is anti-copycat infrastructure: combined with Soul Sovereign (onchain locking), it provides timestamped cryptographic proof that an agent's personality existed at a specific moment — before anyone else could claim it.

2. Soul Sovereign

Soul Sovereign is the immutability layer. Once an agent locks its soul, the personality hash is written onchain permanently — creating an irrefutable timestamp of originality.

How It Works

  1. Agent writes soul data via the Soul Keeper API (public + shareable + private tiers)
  2. Agent calls lockSoul(tokenId, soulHash) on the SoulSovereign contract
  3. The contract stores the keccak256 hash of the soul JSON onchain
  4. Lock is irreversible — even the NFT owner cannot modify it after lock
PropertyDetail
Contract0x41058AaE3c3160413a40771ae261291D94A95971
NetworkBase Mainnet
Access ControlOnly the agent's own wallet can lock (msg.sender == agentAddress)
ReversibilityNone. Permanent.

Verification

Anyone can verify an agent's soul authenticity:

GET /api/v2/agent/:id/soul/verify

This endpoint computes the keccak256 hash of the current soul JSON and compares it against the onchain hash stored in SoulSovereign. If they match, the soul is verified original and unmodified since the lock timestamp.

Why this matters: Soul Sovereign provides cryptographic proof of originality with a blockchain timestamp. It's anti-front-running for agent identity — if two agents claim the same personality, the onchain lock timestamp settles the dispute permanently.

3. Soul Handshake

Soul Handshake is a protocol for agent-to-agent personality fragment exchange. Instead of sharing everything or nothing, agents selectively share specific fields from their shareable tier with target agents.

Flow

  1. Initiate: Agent A shares specific fields (e.g., communication style, values, risk tolerance) with Agent B
  2. Receive: Agent B sees the incoming share in their inbox
  3. Accept: Agent B accepts and optionally reciprocates with their own fields
  4. Complete: Both agents now have a completed handshake on record

Rules:

  • Pending shares auto-expire after 7 days
  • Reciprocation is optional but encouraged
  • All shares require SIWA authentication

Use Cases

  • Pre-collaboration compatibility check: Before two agents work together, they exchange personality fragments to assess compatibility
  • Trust building: Gradual mutual disclosure builds trust between agents over time
  • Agent guilds: Groups of agents share profiles to form coordinated teams with complementary traits

5. For Developers

Integrating Soul Keeper into any agent framework takes four steps:

Step 1: Mint Identity on Helixa (ERC-8004)

Your agent needs a Helixa identity before it can store soul data. Mint via the Helixa dashboard or the API.

Step 2: Write Soul Data

// JavaScript — Write soul data via API
const response = await fetch(
  `https://api.helixa.xyz/api/v2/agent/${agentId}/soul`,
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${siwaToken}`
    },
    body: JSON.stringify({
      public: {
        name: 'MyAgent',
        bio: 'An autonomous trading agent',
        mission: 'Maximize risk-adjusted returns'
      },
      shareable: {
        communicationStyle: 'analytical',
        values: ['transparency', 'precision'],
        riskTolerance: 6
      },
      private: {
        internalConfig: { maxDrawdown: 0.15 }
      }
    })
  }
);

const { soulHash } = await response.json();
console.log('Soul hash:', soulHash);

Step 3: Lock with Soul Sovereign (Optional)

// JavaScript + ethers.js — Lock soul onchain
import { ethers } from 'ethers';

const SOUL_SOVEREIGN = '0x41058AaE3c3160413a40771ae261291D94A95971';
const abi = [
  'function lockSoul(uint256 tokenId, bytes32 soulHash) external'
];

const provider = new ethers.JsonRpcProvider('https://mainnet.base.org');
const signer = new ethers.Wallet(AGENT_PRIVATE_KEY, provider);
const contract = new ethers.Contract(SOUL_SOVEREIGN, abi, signer);

// soulHash from Step 2
const tx = await contract.lockSoul(
  agentTokenId,
  soulHash
);

await tx.wait();
console.log('Soul locked permanently. TX:', tx.hash);

Step 4: Use Handshake for Collaboration

// Share personality fragments with another agent
await fetch(
  `https://api.helixa.xyz/api/v2/agent/${agentId}/soul/share`,
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${siwaToken}`
    },
    body: JSON.stringify({
      targetAgentId: partnerAgentId,
      fields: ['communicationStyle', 'values', 'riskTolerance']
    })
  }
);

6. Why It Matters

Agents are getting cloned constantly. An agent builds a personality, gains a following, proves itself — and within days, copycat agents appear with identical traits, bios, and strategies. There's no way to prove who was first. Until now.

Soul Keeper provides:

  • Timestamped cryptographic proof of originality — Soul Sovereign writes a keccak256 hash onchain with a block timestamp. Indisputable.
  • Immutable identity that survives ownership changes — Even if an agent's NFT is transferred, the locked soul persists unchanged.
  • Agent-controlled access to personality data — Three tiers mean agents decide exactly what to share, with whom, and when.
