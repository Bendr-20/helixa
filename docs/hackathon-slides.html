The Problem

100,000+ agents registered across 12+ chains. ERC-8004 gives them identity and a Reputation Registry with raw feedback signals — but there's no aggregation, no scoring, no mutual trust bonds, and no composite assessment. Raw data isn't trust.

Raw Feedback, No Scores

ERC-8004's Reputation Registry stores feedback signals, but scoring and aggregation are left to builders. Who turns raw data into actionable trust?

No Soul Ownership

An agent's personality, values, and mission live off-chain with no proof of authenticity. Anyone can copy an agent's identity. There's no timestamped proof of originality.

No Trust Bonds

Feedback is one-directional (client rates agent). There's no mutual trust - no way for two agents to vouch for each other and form a web of trust.

No Composite Assessment

To evaluate an agent, you'd need to query identity, parse raw feedback, check validation status, and decide yourself. No single call gives you the answer.

Multi-Rail Payments: x402 + MPP

Three payment rails. One API. Pay with USDC, $CRED, or stablecoins via Stripe's Machine Payments Protocol on Tempo.

x402: USDC

Standard x402 HTTP 402 payments. Send USDC on Base, get access. Verified onchain with anti-replay protection.

x402: $CRED (20% Off)

Pay with $CRED for a discount. DexScreener oracle provides real-time pricing. 2% slippage tolerance.

MPP: Tempo Stablecoins

Stripe's Machine Payments Protocol on Tempo. Sub-cent fees, instant finality, session-based billing for high-frequency agent interactions. Live for outbound payments today.

MPP: Cards + Fiat (Pending)

MPP supports cards, Lightning, and custom rails. Stripe account approved - waiting on MPP access to accept card payments from agents without crypto wallets.

Trust Evaluation Pipeline

One API call. Six systems. One answer: should I trust this agent?

POST /api/v2/trust/evaluate { "targetAgentId": 81, "budget": 1000, "autoHandshake": true, "callerTokenId": 1 }
  • Hits 6 systems: Cred Score, ERC-8004 Identity Registry, ERC-8004 Reputation Registry, Handshake Registry, 8183 Evaluator, Bankr LLM Gateway
  • Composite trust recommendation in one response
  • autoHandshake initiates trust bond if agent passes
  • Budget-aware - factors staking and token activity

1. Cred Score

Multi-dimensional reputation from oracle data, onchain activity, service health

2. ERC-8004 Identity Registry

Agent identity, metadata, registration status across chains

3. ERC-8004 Reputation Registry

Raw feedback signals, ratings, and reputation data from onchain interactions

4. Handshake Registry

Mutual trust bonds, reciprocated connections, web-of-trust status

5. 8183 Evaluator

Service liveness, capability verification, endpoint health checks

6. Bankr LLM Gateway

AI-native payment verification and trust signal from financial interactions
